## Introduction to the Spring IoC Container and Beans

мқҙ мұ•н„°лҠ” мҠӨн”„л§Ғ н”„л Ҳмһ„мӣҢнҒ¬мқҳ IoC к°ңл…җм—җ лҢҖн•ҳм—¬ лӢӨлЈ¬лӢӨ. IoCлҠ” Dependency Injection(DI)лЎңлҸ„ м•Ңл Өм ё мһҲлӢӨ. мқҙкІғмқҖ к°қмІҙк°Җ мӮ¬мҡ©н•ҳлҠ” мқҳмЎҙм„ұмқ„ Constructor Arguments, Arguments to a factory method, Properties, лҰ¬н„ҙлҗң к°қмІҙ мқёмҠӨн„ҙмҠӨм—җ м„Өм •лҗң нҠ№м„ұмңјлЎңл§Ң м •мқҳн•ҳкі  мІҳлҰ¬н•ҳлҸ„лЎқ н•ңлӢӨ. м»Ён…ҢмқҙлҠ” л№Ҳмқ„ мғқм„ұн•  л•Ң н•„мҡ”н•ң мқҳмЎҙм„ұмқ„ мЈјмһ…н•ҳкі  мқҙлҹ¬н•ң н”„лЎңм„ёмҠӨлҠ” Service Locator pattern л©”м»ӨлӢҲмҰҳ лҳҗлҠ” м§Ғм ‘м Ғмқё нҒҙлһҳмҠӨмқҳ мғқм„ұмһҗлҘј мӮ¬мҡ©н•ЁмңјлЎңмҚЁ мқҳмЎҙм„ұмқҳ мң„м№ҳмҷҖ мқёмҠӨн„ҙмҠӨнҷ”лҘј мҠӨмҠӨлЎң л№Ҳмқҙ м»ЁнҠёлЎӨн•  мҲҳ мһҲлҠ” inverse мқҙлӢӨ.

`org.springframework.beans`кіј `org.springframework.context` нҢЁнӮӨм§ҖлҠ” мҠӨн”„л§Ғ н”„лҰ¬м— мӣҢнҒ¬ IoC м»Ён…Ңмқҙл„Ҳмқҳ н•өмӢ¬ нҢЁнӮӨм§Җмқҙкі  `BeanFactory` мқён„°нҺҳмқҙмҠӨлҠ” лӘЁл“  нғҖмһ…мқҳ к°қмІҙлҘј кҙҖлҰ¬н•  мҲҳ мһҲлҠ” л§Өм»ӨлӢҲмҰҳм—җ лҢҖн•ң нҷ•мһҘлҗң нҷҳкІҪм„Өм •мқ„ н•  мҲҳ мһҲлӢӨ. `ApplicationContext`лҠ” `BeanFactory`мқҳ sub-interface мқҙлӢӨ. м¶”к°ҖлЎң

* мҠӨн”јл§Ғ AOP кё°лҠҘл“Өмқ„ мүҪкІҢ integration н•  мҲҳ мһҲмқҢ.
* internationalization к°ҷмқҖ л©”м„ём§Җ лҰ¬мҶҢмҠӨлҘј н•ёл“Өл§Ғн•  мҲҳ мһҲмқҢ.
* Event publication
* мӣ№ м• н”ҢлҰ¬мјҖмқҙм…ҳмқ„ мң„н•ң `WebApplicationContext` к°ҷмқҖ м• н”ҢлҰ¬мјҖмқҙм…ҳ л Ҳмқҙм–ҙлҘј мң„н•ң нҠ№м • мҪҳн…ҚмҠӨнҠёлҘј м ңкіө

м •лҰ¬н•ҳмһҗл©ҙ `BeanFactory`лҠ” кё°ліём Ғмқё н”„л Ҳмһ„мӣҢнҒ¬мқҳ кё°лҠҘкіј нҷҳкІҪм„Өм •мқ„ н•  мҲҳмһҲлҸ„лЎқ м ңкіөн•ҳл©° `ApplicationContext`лҠ” м—”н„°н”„лқјмқҙмҰҲ кё°лҠҘмқ„ мң„н•ң лҚ” л§ҺмқҖ кё°лҠҘмқ„ м§Җмӣҗн•ңлӢӨкі  м •лҰ¬н•  мҲҳ мһҲлӢӨ. `ApplicationContext`лҠ” `BeanFactory`мқҳ supersetмқҙл©° мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„ҲлҘј м„ӨлӘ…н•ҳлҠ” мқҙ мұ•н„°м—җм„ңл§Ң мӮ¬мҡ©лҗңлӢӨ.

мҠӨн”„л§Ғм—җм„ң м• н”ҢлҰ¬мјҖмқҙм…ҳмқҳ мӨ‘м¶”м—җм„ң мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„Ҳм—җ мқҳн•ҙм„ң кҙҖлҰ¬лҗҳлҠ” к°қмІҙлҘј л№Ҳ(bean)мқҙлқјкі  л¶ҖлҘёлӢӨ. л№ҲмқҖ мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„Ҳм—җ мқҳн•ҙм„ң мЎ°лҰҪ/мқёмҠӨн„ҙмҠӨнҷ” лҗҳкі  кҙҖлҰ¬лҗңлӢӨ. л№ҲмқҖ м• н”ҢлҰ¬мјҖмқҙм…ҳм—җм„ң л§Һмқ„ мҲҳ мһҲмңјл©° л№Ҳкіј л№Ҳ кҙҖл Ё мқҳмЎҙм„ұ(Dependency)лҠ” м»Ён…Ңмқҙл„Ҳмқҳ нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°м—җ мқҳн•ҙ м„Өм •лҗңлӢӨ.

## Container Overview

`org.springframework.context.ApplicationContext` мқён„°нҺҳмқҙмҠӨлҠ” мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„ҲлҘј лҢҖн‘ңн•ҳкі  л№Ҳмқҳ мқёмҠӨн„ҙмҠӨ, м„Өм •, мЎ°лҰҪм—җ лҢҖн•ң мұ…мһ„мқ„ к°Җм§„лӢӨ. м»Ён…Ңмқҙл„ҲлҠ” нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°лҘј мқҪм–ҙм„ң к°қмІҙлҘј м„Өм •н•ҳкі , мЎ°лҰҪн•ҳкі , мқёмҠӨн„ҙмҠӨнҷ”мӢңнӮЁлӢӨ. к·ё нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°лҠ” XML, Java Annotation, Java Code л“ұмңјлЎң мһ‘м„ұлҗ  мҲҳ мһҲмңјл©° мқҙлҘј нҶөн•ҳм—¬ к°ңл°ңмһҗлҠ” к°қмІҙк°„мқҳ мғҒнҳё мқҳмЎҙм„ұмқ„ м„Өм •н•ҳкі  м• н”ҢлҰ¬мјҖмқҙм…ҳмқ„ л§Ңл“Ө мҲҳ мһҲлҠ” н‘ңнҳ„мқ„ н•  мҲҳ мһҲлҸ„лЎқ н•ңлӢӨ.

лӘҮлӘҮ `ApplicationContext` мқён„°нҺҳмқҙмҠӨмқҳ кө¬нҳ„мІҙлҠ” мҠӨн”„л§Ғм—җ мқҳн•ҙм„ң м ңкіөлҗңлӢӨ. лҸ…лҰҪнҳ•(Stand-alone) м• н”ҢлҰ¬мјҖмқҙм…ҳм—җм„ң мқјл°ҳм ҒмңјлЎң `ClassPathXmlApplicationContext`, `FileSystemXmlApplicationContext`к°Җ л§Һмқҙ мӮ¬мҡ©лҗҳл©°, лӘЁл“  л©”нғҖлҚ°мқҙн„° м„Өм •мқ„ м „нҶөм Ғмқё нҸ¬л§·мқё XMLлЎңлҸ„ м„Өм •н•  мҲҳ мһҲм§Җл§Ң, XMLм—җ м• л…ён…Ңмқҙм…ҳ лҳҗлҠ” мҪ”л“ң кё°л°ҳмқҳ м„Өм •мқ„ н• кІғмқҙлқјкі  лӘҮ мӨ„л§Ң лӘ…мӢңн•ҳкі  лӮҳлЁём§ҖлҠ” мқҙлҹ¬н•ң м„Өм •м—җ л”°лқј м• л…ён…Ңмқҙм…ҳ лҳҗлҠ” мҪ”л“ңлЎң м„Өм •н•  мҲҳлҸ„ мһҲлӢӨ.

лҢҖл¶Җл¶„мқҳ м• н”Ңл ҲмјҖмқҙм…ҳ мӢңлӮҳлҰ¬мҳӨм—җм„ң к°ңл°ңмһҗк°Җ мҪ”л“ңлЎң н•ҳлӮҳ лҳҗлҠ” к·ё мқҙмғҒмқҳ мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„ҲлҘј мқёмҠӨн„ҙмҠӨнҷ” мӢңнӮ¬ н•„мҡ”лҠ” м—ҶлӢӨ. мҳҲлҘјл“Өм–ҙ мӣ№ м• н”ҢлҰ¬мјҖмқҙм…ҳмқ„ м„Өм •н• л•Ң к°„лӢЁн•ң 8мӨ„мқҳ boilerpalte web descriptor XML кө¬л¬ёмқ„ `web.xml` нҢҢмқјм—җ м¶”к°Җн•ҳл©ҙ м• н”ҢлҰ¬мјҖмқҙм…ҳмқ„ кө¬м„ұн•ҳлҠ”лҚ° м¶©л¶„н•ҳлӢӨ.  л§Ңм•Ҫм—җ STS(Spring Tool Suite)лҘј мӮ¬мҡ©н•ңлӢӨл©ҙ мқҙлҹ¬н•ң м„Өм •мқ„ л§Ҳмҡ°мҠӨ нҒҙлҰӯл§ҢмңјлЎң мүҪкІҢ мғқм„ұн•  мҲҳ мһҲлӢӨ.

м•„лһҳмқҳ лӢӨмқҙм–ҙк·ёлһЁмқҖ мҠӨн”„л§Ғмқҙ м–ҙл–»кІҢ лҸҷмһ‘н•ҳлҠ”м§Җ к°„лӢЁн•ҳкІҢ ліҙм—¬мӨҖлӢӨ. мӮ¬мҡ©мһҗ м• н”ҢлҰ¬мјҖмқҙм…ҳмқҳ нҒҙлһҳмҠӨл“ӨмқҖ нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°мҷҖ н•Ёк»ҳ кө¬м„ұлҗҳкі  `ApplicationContext`к°Җ мғқм„ұлҗҳкі  мҙҲкё°нҷ”лҗҳл©ҙ мӮ¬мҡ©мһҗлҠ” мҷ„лІҪн•ҳкІҢ м„Өм •лҗҳкі  мӢӨн–үк°ҖлҠҘн•ң м• н”ҢлҰ¬мјҖмқҙм…ҳ лҳҗлҠ” мӢңмҠӨн…ңмқ„ к°Җм§Ҳ мҲҳ мһҲкІҢ лҗңлӢӨ.

### Configuration Metadata

мқҙм „мқҳ лӢӨмқҙм–ҙк·ёлһЁм—җм„ң лҙӨл“Ҝмқҙ мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„ҲлҠ” нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°м—җ мқҳн•ҙм„ң кө¬м„ұлҗңлӢӨ. нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°лҠ” к°ңл°ңмһҗк°Җ мҠӨн”„л§Ғ м»Ён…Ңмқҙл„ҲлҘј м–ҙл–»кІҢ м„Өм •н•ҳкі , кө¬м„ұн•ҳкі  мқёмҠӨн„ҙмҠӨнҷ”н•  кІғмқём§Җм—җ лҢҖн•ҙ кё°мҲ лҗҳм–ҙ мһҲлӢӨ.

нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°лҠ” м „нҶөм ҒмңјлЎң XML нҸ¬л§·мңјлЎң м„Өм •н•  мҲҳ мһҲлҸ„лЎқ м§Җмӣҗн•ҳл©° ліё л¬ём„ңм—җм„ң мҠӨн”„л§Ғ IoC м»Ён…Ңмқҙл„Ҳмқҳ н•өмӢ¬ м„Өм •кіј м»Ём…үмқ„ м„ӨлӘ…н•ҳлҠ”лҚ° XMLмқ„ мӮ¬мҡ© н•  кІғмқҙлӢӨ.

м•„лһҳмҷҖ к°ҷмқҖ нҸ¬л§·лҸ„ м§Җмӣҗн•ңлӢӨ.

* Annotation-based configuration : мҠӨн”„л§Ғ 2.5 л¶Җн„° м§Җмӣҗн•ҳмҳҖлӢӨ.
* Java-based configuration : мҠӨн”„л§Ғ 3.0 мӢңмһ‘н•ҳмҳҖмңјл©° Spring Java Config н”„лЎңм қнҠёк°Җ м§Җкіөн•ҳлҠ” л§ҺмқҖ кё°лҠҘл“Өмқҙ мҠӨн”„л§Ғ н”„л Ҳмһ„мӣҢнҒ¬мқҳ мқјл¶Җк°Җ лҗҳм—ҲлӢӨ. л”°лқјм„ң мӮ¬мҡ©мһҗлҠ” м• н”ҢлҰ¬мјҖмқҙм…ҳмқҳ мҷёл¶Җм—җм„ң XML нҢҢмқј лҢҖмӢ  мһҗл°”лҘј мӮ¬мҡ©н•ҳм—¬ м• н”ҢлҰ¬мјҖмқҙм…ҳ нҒҙлһҳмҠӨ мҷёл¶Җмқҳ л№Ҳмқ„ м •мқҳн•  мҲҳ мһҲлӢӨ.

мҠӨн”„л§Ғ м„Өм •мқҖ мқјл°ҳм ҒмңјлЎң м»Ён…Ңмқҙл„Ҳк°Җ кҙҖлҰ¬н•ҙм•ј н•ҳлҠ” мөңмҶҢ н•ҳлӮҳ мқҙмғҒмқҳ л№Ҳ м„Өм •мңјлЎң мқҙлЈЁм–ҙм ё мһҲлӢӨ. XML кё°л°ҳмқҳ м„Өм • л©”нғҖлҚ°мқҙн„°лҠ” `<beans/>` нғңк·ё м•Ҳмқҳ `<bean/>` нғңк·ём—җ л№Ҳл“Өмқ„ кө¬м„ұн•ңлӢӨ. мһҗл°” м„Өм •мқҖ мқјл°ҳм ҒмңјлЎң `@Configuration`мқҙ м„ м–ёлҗң нҒҙлһҳмҠӨмқҳ `@Bean` м• л…ён…Ңмқҙм…ҳ л©”мҶҢл“ңлЎң м„Өм •лҗңлӢӨ.

мқҙлҹ¬н•ң л№Ҳл“Өмқҳ м •мқҳлҠ” мӢӨм ң мӮ¬мҡ©мһҗ м• н”ҢлҰ¬мјҖмқҙм…ҳмқ„ л§Ңл“ңлҠ” к°қмІҙлЎң кө¬м„ұлҗҳм–ҙ мһҲлӢӨ. мқјл°ҳм ҒмңјлЎң мӮ¬мҡ©мһҗлҠ” Service Layer Objects, Data Access Objects(DAOs), Presentation Objects(ex. Strutsмқҳ `Action`), Infrastructure Objects(ex. Hibernateмқҳ `SessionFactories`), JMS `Queues` л“ұмқҳ м„ңл№„мҠӨлҘј м •мқҳн•ңлӢӨ. Typically, one does not configure fine-grained domain objects in the container, because it is usually the responsibility of DAOs and business logic to create and load domain objects. However, you can use SpringвҖҷs integration with AspectJ to configure objects that have been created outside the control of an IoC container. See [Using AspectJ to dependency-inject domain objects with Spring](https://docs.spring.io/spring/docs/5.1.8.RELEASE/spring-framework-reference/core.html#aop-atconfigurable).

м•„лһҳлҠ” XML кё°л°ҳмқҳ кё°ліё нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„° кө¬мЎ°мқҙлӢӨ.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="..." class="...">   
        <!-- collaborators and configuration for this bean go here -->
    </bean>
    <bean id="..." class="...">
        <!-- collaborators and configuration for this bean go here -->
    </bean>
    <!-- more bean definitions go here -->
</beans>
```

мң„ м„Өм •м—җм„ң `id` мҶҚм„ұмқҖ к°Ғк°Ғмқҳ л№Ҳ м •мқҳлҘј кө¬л¶„н•ҳкё° мң„н•ң л¬ёмһҗм—ҙ мҶҚм„ұмқҙл©° к°қмІҙк°„м—җ м„ңлЎңлҘј м°ёмЎ°н•  л•Ң мӮ¬мҡ©лҗңлӢӨ. `class` мҶҚм„ұмқҖ л№Ҳмқҳ нғҖмһ…мқ„ м •мқҳн•ҳкё° мң„н•ң fully qualified classname(нҢЁнӮӨм§Җ лӘ…мқ„ нҸ¬н•Ён•ң н’Җ нҒҙлһҳмҠӨ л„Өмһ„) мқҙлӢӨ.

### Instantiating a Container

`ApplicationContext`лҘј м ңкіөн•ҳкё° мң„н•ң location path мғқм„ұмһҗлҠ” м»Ён…Ңмқҙл„Ҳк°Җ м—¬лҹ¬ мҷёл¶Җ лҰ¬мҶҢмҠӨл“Ө(local file system лҳҗлҠ” Java `CLASSPATH`)лЎң л¶Җн„° нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„°лҘј мқҪлҸ„лЎқ н•ҙмӨҖлӢӨ.

```java
ApplicationContext context = new ClassPathXmlApplicationContext("services.xml", "daos.xml");
```

#### Composing XML-based Configuration Metadata

л№Ҳ м„Өм •мқ„ м—¬лҹ¬ XML нҢҢмқјм—җ н•ҳлҠ” кІғмқҖ мң мҡ©н•ңлӢӨ. мў…мў… к°ңлі„ XML м„Өм • нҢҢмқјмқ„ м•„нӮӨн…Қміҗмқҳ лӘЁл“Ҳ лҳҗлҠ” л…јлҰ¬м Ғ л Ҳмқҙм–ҙлҘј н‘ңнҳ„н•ңлӢӨ. к°ңл°ңмһҗлҠ” лӘЁл“  XML fragmentл“ӨлЎңл¶Җн„° л№Ҳ м„Өм •мқ„ мқҪкё°мң„н•ҙ м• н”ҢлҰ¬мјҖмқҙм…ҳ мҪҳн…ҚмҠӨнҠё мғқм„ұмһҗ(application context construct)лҘј мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.  м•„лһҳмқҳ мғқм„ұмһҗлҠ” м—¬лҹ¬ `Resource`лҘј мӮ¬мҡ©н•ңлӢӨ. м•„лһҳ мҳҲм ңлҠ” м—¬лҹ¬ нҢҢмқјмқҳ bean м„Өм •мқ„ лЎңл“ңн•ҳл Өкі  `<import/>` м—ҳлҰ¬лЁјнҠёлҘј мӮ¬мҡ©н•ң мҳҲм ңмқҙлӢӨ. 

```xml
<beans>
    <import resource="services.xml"/>
    <import resource="resources/messageSource.xml"/>
    <import resource="/resources/themeSource.xml"/>

    <bean id="bean1" class="..."/>
    <bean id="bean2" class="..."/>
</beans>
```

мқҙ мҳҲм ңм—җм„ң мҷёл¶Җ л№Ҳ м •мқҳлҠ” `services.xml, messageSource.xml, themeSource.xml`м—җ мқҳн•ҙм„ң лЎңл”©лҗңлӢӨ. лӘЁл“  лЎңмјҖмқҙм…ҳмқҳ кІҪлЎңлҠ” м •мқҳн•ң нҢҢмқјмқ„ importн•ң мң„м№ҳлҘј кё°мӨҖмңјлЎң н•ҳлҠ” мғҒлҢҖкІҪлЎңлЎң importлҗҳл©° мқҙ мҳҲм ңмқҙм„ң `services.xml`мқҖ мқҙ л№Ҳ м„Өм • нҢҢмқјкіј к°ҷмқҖкіі л””л үнҶ лҰ¬ лҳҗлҠ” нҒҙлһҳмҠӨнҢЁмҠӨм—җ мң„м№ҳн•ҙ мһҲмқҢмқ„ мқҳлҜён•ңлӢӨ. лҳҗн•ң `messageSource.xml, themeSource.xml`мқҖ  л°ҳл“ңмӢң importн•ң мң„м№ҳмқҳ `resources` нҸҙлҚ” м•Ҳм—җ мһҲм–ҙм•ј н•ңлӢӨ. м—¬кё°м„ң л§Ё м•һм—җ лӮҳмҳӨлҠ” `/`лҠ” л¬ҙмӢңлҗңлӢӨ. к·ёлҹ¬лӮҳ м—¬кё°м„ң мӮ¬мҡ©лҗҳлҠ” кІҪлЎңк°Җ мғҒлҢҖкІҪлЎң мқј кІҪмҡ° л§Ё м•һм—җ мҳӨлҠ” `/`лҠ” м“°м§Җ м•ҠлҠ” кІғмқҙ мўӢлӢӨ. `<beans/>` м—ҳлҰ¬лЁјнҠёлҘј нҸ¬н•Ён•ҳкі  мһҲлҠ” importлҗҳм–ҙм ё мһҲлҠ” нҢҢмқјмқҳ лӮҙмҡ©мқҖ  Spring Schemaмқҳ XML bean definition л¬ёлІ•м—җ л§һм•„м•ј н•ңлӢӨ.

> It is possible, but not recommended, to reference files in parent directories using a relative "../" path. Doing so creates a dependency on a file that is outside the current application. In particular, this reference is not recommended for `classpath:` URLs (for example, `classpath:../services.xml`), where the runtime resolution process chooses the вҖңnearestвҖқ classpath root and then looks into its parent directory. Classpath configuration changes may lead to the choice of a different, incorrect directory. 

>You can always use fully qualified resource locations instead of relative paths: for example, `file:C:/config/services.xml` or `classpath:/config/services.xml`. However, be aware that you are coupling your applicationвҖҷs configuration to specific absolute locations. It is generally preferable to keep an indirection for such absolute locationsвҖүвҖ”вҖүfor example, through "${вҖҰвҖӢ}" placeholders that are resolved against JVM system properties at runtime.

The namespace itself provides the import directive feature. Further configuration features beyond plain bean definitions are available in a selection of XML namespaces provided by SpringвҖүвҖ”вҖүfor example, the `context`and `util` namespaces.

#### The Groovy Bean Definition DSL

 мҷёл¶Җнҷ”лҗң нҷҳкІҪм„Өм • л©”нғҖлҚ°мқҙн„° м„Өм •мқ„ мң„н•ң мқҙм „мқҳ мҳҲм ң мІҳлҹј, л№Ҳ м„Өм •мқ„ Grails frameworkлЎң л¶Җн„° нҢҢмғқлҗң мҠӨн”„л§Ғмқҳ к·ёлЈЁл№„ л№Ҳ м„Өм • DSL(Groovy Bean Definition DSL)лЎңлҸ„ н•  мҲҳ мһҲлӢӨ. ліҙнҶө `".groovy"` нҢҢмқјлЎң м•„лһҳмқҳ мҳҲм ңлҘј нҶөн•ҳм—¬ м„Өм •н•  мҲҳ мһҲлӢӨ.

```groovy
beans {
    dataSource(BasicDataSource) {
        driverClassName = "org.hsqldb.jdbcDriver"
        url = "jdbc:hsqldb:mem:grailsDB"
        username = "sa"
        password = ""
        settings = [mynew:"setting"]
    }
    sessionFactory(SessionFactory) {
        dataSource = dataSource
    }
    myService(MyService) {
        nestedBean = { AnotherBean bean ->
            dataSource = dataSource
        }
    }
}
```

мқҙ м„Өм •мқҖ XML bean definitionкіј кұ°мқҳ лҸҷмқјн•ҳл©° мҠӨн”„л§Ғмқҳ XML нҷҳкІҪм„Өм • л„Өмһ„мҠӨнҺҳмқҙмҠӨлҸ„ м§Җмӣҗн•ңлӢӨ. лҳҗн•ң `importBean` directiveлҘј мӮ¬мҡ©н•ҙм„ң XML л№Ҳ м„Өм •мқ„ мһ„нҸ¬нҠён• мҲҳлҸ„ мһҲлӢӨ.

### Using the Container

`ApplicationContext`лҠ” к°Ғк°Ғмқҳ л№Ҳл“Өмқ„ мғқм„ұ, мқҳмЎҙм„ұмқ„ кҙҖлҰ¬н•ҳлҠ” кё°лҠҘмқ„ к°Җм§„ advancded factoryлҘј мң„н•ң мқён„°нҺҳмқҙмҠӨмқҙлӢӨ. `T getBean(String name, Class<T> requiredType)`л©”мҶҢл“ңлҘј мӮ¬мҡ©н•ЁмңјлЎңмҚЁ к°ңл°ңмһҗлҠ” л№Ҳмқҳ мқёмҠӨн„ҙмҠӨлҘј м–»мқ„ мҲҳ мһҲлӢӨ.

`ApplicationContext`лҠ” к°ңл°ңмһҗк°Җ л№Ҳ м •мқҳлҘј мқҪкі  м ‘мҶҚн•  мҲҳ мһҲлҸ„лЎқ н•ҙмӨҖлӢӨ.

```java
// create and configure beans
ApplicationContext context = new ClassPathXmlApplicationContext("services.xml", "daos.xml");

// retrieve configured instance
PetStoreService service = context.getBean("petStore", PetStoreService.class);

// use configured instance
List<String> userList = service.getUsernameList();
```

к·ёлЈЁл№„ нҷҳкІҪм„Өм •лҸ„ л§Өмҡ° мң мӮ¬н•ҳкІҢ мІҳлҰ¬н•  мҲҳ мһҲлӢӨ. к·ёлЈЁл№„лҘј мң„н•ң лі„лҸ„мқҳ м»Ён…ҚмҠӨнҠё кө¬нҳ„мІҙк°Җ мһҲлӢӨ.

```java
ApplicationContext context = new GenericGroovyApplicationContext("services.groovy", "daos.groovy");
```

к°ҖмһҘ мң м—°н•ң ліҖнҳ•мқҖ `GenericApplicationContext` мқҙлӢӨ. мҳҲлҘјл“Өм–ҙ XML нҢҢмқјмқ„ мқҪкё° мң„н•ҙм„ң `XmlBeanDefinitionReader` мҷҖ н•Ёк»ҳ мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.

```java
GenericApplicationContext context = new GenericApplicationContext();

```
