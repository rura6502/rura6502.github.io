# What is Immutable Class

변경 불가능한 객체. heap 영역에 생성됨. 자바에선 대표적으로 String, Boolean, Integer, Float, Long 등이 있음. 이 불변 객체는 값을 변경할 수 없고 새로운 객체로 대체됨.

특징

* 장점 : 생성자, 접근메소드에 대한 방어 복사가 필요 없음. 멀티 스레드 환경에서 안전함, 동기화 처리 없이 객체 공유 가능.
* 단점 : 객체 생성 비용으로 인해서 메모리 낭비, 성능 저하 발생.

멀티스레딩환경에서의 문제

특정 객체를 멀티스레드가 접근하여 사용할 때 한 스레드에서 객체에 상태변화를 할 경우 모든 스레드가 변화된 객체로 접근하게 되어 예측하지 못한 문제가 발생할 수 있다. 불변객체의 경우 상태 변경 시 기존의 객체가 변경되는 것이 아니라 새로운 메모리를 참조하는 새로운 객체를 반환하므로 다른 스레드는 다른 스레드가 변화시키기 전의 객체를 사용하여 상태변화에 영향을 받지 않는다.
