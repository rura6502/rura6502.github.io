# 객체지향 프로그램 설계 원칙
## KISS, Keep it short and simple
코드는 항상 명확하고 깔끔하게 작성해라. 복잡하고 간결하지 않은 코드는 상대방을 이해시키기 어렵고 내일의 나조차도 보기 힘들다.
* Bad
	 ```java
	 public String getDayString(int sequence) {
	   if (sequence == 0) {
		   return "Sunday";
	   } else if (sequence == 1) {
		   return "Monday";
	   } else if (sequence == 1) {
		   return "Tuesday";
	   } else if (sequence == 1) {
		   return "Wednesday";
	   } else if (sequence == 1) {
		   return "Thursday";
	   } else if (sequence == 1) {
		   return "Friday";
	   } else if (sequence == 1) {
		   return "Saturday";
	   } else {
		   return "unknown";
	   }
	 } 
   ```
 * Good
    ```java
    asdfasdf
  ```
