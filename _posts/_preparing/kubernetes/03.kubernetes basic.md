쿠버넷의 구성요소

* etcd
* 쿠버넷 마스터 : HTTP/HTTPS로 etcd와 연결되어 데이터를 저장
* 쿠버넷 노드 : HTTP/HTTPS로 마스터와 연결되어 명령을 수신하고 상태를 보고
* 쿠버넷 네트워크 : 컨테이너 애플리케이션의 연결을 구성

### Kubernetes Mater

대표적인 기능

* 인증 및 인가
* RESTful API 엔트리 포인트
* 컨테이너 배포 스케쥴러
* 확장 및 복제 컨트롤러
* 클러스터 구성 환경 읽기

#### API 서버(kube-apiserver)

HTTP/HTTPS 기반 RESTful API 제공. 쿠버넷 구성 요소 간 허브 역할을 함

#### 스케줄러(kube-scheduler)

컨테이너가 어떤 노드에서 동작 할 것인지에 대한 선택 기능을 지원. 컨테이너를 노드에 디스패치 및 연결하기 위한 우선순위를 결정하는 단순 알고리즘. CPU, 메모리, 동작 중인 컨테이너의 수가 사용 됨.

#### 컨트롤러 관리자(kube-controller-manager)

클러스터의 운영 기능을 수행. 대표적인 기능

* 쿠버넷 노드 관리
* 쿠버넷 내부 정보 생성 및 업데이터
* 기대하는 상태로의 변경 시도

#### 명령줄 인터페이스(kubectl)

쿠버넷 클러스터 제어 인터페이스.

### Kubernetes Node

쿠버넷 클러스터의 슬레이브 노드. 도커나 rkt를 사용해 컨테이너 애플리케이션을 실행.

#### kubelet

노드의 주요 프로세스 중 하나. 마스터와 통신하여 아래의 작업을 수행

* 확인 및 보고를 위하여 주기적으로 API 컨트롤러에 접근
* 컨테이너 운영
* HTTP 서버로써 API 제공

#### 프록시(kube-proxy)

컨테이너를 위한 네트워크 프록세와 로드 밸런서를 담당.

### etcd

쿠버넷이 사용하는 분산 key-value 형태의 데이터 저장소. RESTful API를 사용하여 접근할 수 있으며 네트워크를 통하여 CRUD

```bash
// 쿠버넷 구성과 상태를 확인할 수 있음
curl -L http://127.0.0.1:4001/v2/keys/registry
```
