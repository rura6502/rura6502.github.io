## 도커 기초

### Dockerfile

#### FROM

빌드할 때 사용하는 베이스 이미지를 의미. 도커 모범 사례에서는 Debian을 기본 리눅스 배포판으로 사용하도로 권장. Node.js나 파이썬과 같은 언어를 지원하도록 특별히 제작된 컨테이너도 있음. https://hub.docker.com/explore 에서 다양한 종류의 베이스 이미지를 확인할 수 있음.

#### RUN

이미지 빌드 과정에서 실행하는 커맨드를 의미. 베이스 이미지에서 디펜던시나 기타 라이브러리를 추가하는데 가장 많이 사용. (예. `apt-get`, `rpm -ivh` 등)

#### ENV

컨테이너 내부에서 사용할 환경변수를 설정하는 역할.

#### COPY

`ADD` 라고도 불리며 로컬 파일을 복사하는데 사용.

#### LABE

`docker inspect` 커맨드를 통해서 볼 수 있는 값을 설정. 베이스 이미지를 바탕으로 계속 추가 되는 형태임.

#### CMD, ENDPOINT

이미지를 실행할 때 함꼐 실행해야 할 커맨드와 스크립트 등을 명시.

### Dockerfile의 목적

* 기본이 되는 OS의 보안 패치 다운로드와 설치
* 코드 실행에 필요한 프로그래밍 언어와 실행 환경 설치
* 코드 실행에 필요한 디펜던시 설치
* 코드를 컨테이너 내부로 복사
* 실행 대상 및 실행 방법 정의

### 도커 이름 구조

`[레지스트리 주소]/[컨테이너 네임스페이스]/[컨테이너 이름]:[태그]

### 도커 주요 명령어

* `docker build .` : 현재 디렉토리(.) 기준으로 `Dockerfile`을 읽어서 이미지를 빌드
	* `-t` : 태그를 같이 부여
* `docker tag` : 생성한 이미지에 태그를 부여
	* `docker tag 42f37 [IMAGE_NAME]:[TAG]`
* `docker login [REGISTRY]` : 레지스트리(컨테이너 저장소) 로그인
* `docker push` : 레지스트리에 이미지 푸쉬
* `docker images` : 현재 ㅗ커
